## ภาพรวม (Overview)
เอกสารชุดนี้ใช้สำหรับ **ยกระดับต้นแบบ PM Master System ให้เป็นเว็บแอปจริงที่เก็บข้อมูลส่วนกลาง** —
ทุกคนในทีมเห็นข้อมูลชุดเดียวกัน, มีระบบ login, และเข้าใช้งานได้จากทุกอุปกรณ์ผ่าน URL

ต้นแบบปัจจุบัน (ไฟล์ `PM Master System.dc.html`) ทำงานได้ครบทุกหน้าจอแล้ว แต่เก็บข้อมูลไว้ใน
เบราว์เซอร์ของแต่ละเครื่อง (localStorage) เท่านั้น เอกสารนี้อธิบายวิธีแทนที่ด้วย **Supabase**
(ฐานข้อมูล PostgreSQL + Auth + Storage แบบ hosted ที่มีแพ็กเกจฟรี)

> **สำคัญ:** ไฟล์ HTML ในแพ็กเกจนี้คือ **ต้นแบบอ้างอิงด้านดีไซน์และพฤติกรรม** ไม่ใช่โค้ดที่จะนำขึ้น
> production ตรงๆ งานของนักพัฒนาคือ "สร้างหน้าจอตามต้นแบบนี้ขึ้นใหม่ในสภาพแวดล้อมจริง"
> (แนะนำ **React + Vite** หรือ **Next.js**) แล้วต่อชั้นข้อมูลตามไฟล์ในแพ็กเกจ

## ขั้นตอนการติดตั้ง (ทำตามนี้ทีละข้อ)

### ขั้นที่ 1 — สร้างโปรเจกต์ Supabase
1. สมัคร/เข้าสู่ระบบที่ **supabase.com** → **New project**
2. ตั้งชื่อ เช่น `pm-kasetchon`, ตั้งรหัสผ่านฐานข้อมูล, เลือก region **Singapore** (ใกล้ไทยสุด)
3. รอสร้างเสร็จ (~2 นาที)

### ขั้นที่ 2 — สร้างตาราง
1. เปิดเมนู **SQL Editor → New query**
2. คัดลอกเนื้อหาทั้งหมดจาก `01_supabase_schema.sql` ไปวาง → กด **Run**
3. ไปที่ **Table Editor** จะเห็นตาราง `machines`, `pm_plans`, `work_orders`, `wo_checklist`, `wo_comments`, `profiles` พร้อมข้อมูลตัวอย่าง

### ขั้นที่ 3 — เปิด Storage สำหรับรูป Before/After
1. เมนู **Storage → New bucket** → ตั้งชื่อ `wo-photos` → เลือก **Public**
2. (รูปที่อัปโหลดในใบสั่งงานจะถูกเก็บที่นี่)

### ขั้นที่ 4 — เปิด Authentication
1. เมนู **Authentication → Providers** → เปิด **Email** (ค่าเริ่มต้นเปิดอยู่แล้ว)
2. สร้างผู้ใช้ทดสอบที่ **Authentication → Users → Add user**

### ขั้นที่ 5 — ดึงคีย์มาใส่โค้ด
1. เมนู **Project Settings → API**
2. คัดลอก **Project URL** และ **anon public key**
3. สร้างไฟล์ `.env` ในโปรเจกต์เว็บ:
   ```
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGci...
   ```
4. ติดตั้งไลบรารี: `npm install @supabase/supabase-js`

### ขั้นที่ 6 — ต่อชั้นข้อมูลเข้ากับหน้าจอ
นำ `02_pmDataLayer.js` เข้าโปรเจกต์ แล้วแทนที่จุดที่ต้นแบบใช้ localStorage ด้วยฟังก์ชันจากไฟล์นี้
(ตารางแมปด้านล่าง)

## แผนผังข้อมูล (Data Model)
```
profiles      ── ผู้ใช้ (เชื่อมกับ auth.users)
machines      ── เครื่องจักร 325 เครื่อง
pm_plans      ── แผนบำรุงรักษา (1 เครื่อง มีได้หลายแผน)
   └─ work_orders   ── ใบสั่งงานที่ออกจากแผน
        ├─ wo_checklist  ── รายการตรวจสอบในใบสั่งงาน
        └─ wo_comments   ── ความคิดเห็น/บันทึก + รูป before/after
```

## จุดเชื่อมต่อ: localStorage เดิม → ฟังก์ชันใหม่
| การทำงานในต้นแบบ | state เดิม | แทนด้วย |
|---|---|---|
| โหลดเครื่องจักร | `machineMeta()` + `extraMachines` | `db.listMachines()` |
| เพิ่มเครื่องจักร | `saveMachine` | `db.addMachine(...)` |
| ลบเครื่องจักร | `deletedMachines` | `db.deleteMachine(id)` |
| โหลดแผน PM | `planBase` + `extraPlans` | `db.listPlans()` |
| สร้าง/แก้ไขแผน | `savePm` / `planOverrides` | `db.addPlan()` / `db.updatePlan()` |
| พัก/เปิดใช้แผน | `togglePause` | `db.setPlanPaused(id, bool)` |
| ส่งใบสั่งงานอัตโนมัติ 5 วัน | คำนวณใน `renderVals` | `autoDispatchDueWork()` |
| ติ๊ก checklist | `checks` | `db.toggleChecklist(itemId, bool)` |
| เพิ่มความคิดเห็น | `comments` | `db.addComment(...)` |
| อัปโหลดรูป before/after | `photos` (data URL) | `db.uploadPhoto(woId, kind, file)` |
| login / ออกจากระบบ | (ยังไม่มี) | `auth.signIn()` / `auth.signOut()` |

## หน้าจอในต้นแบบ (Screens) — สร้างตามนี้
1. **แดชบอร์ด** — KPI 6 ใบ, กราฟแนวโน้ม PM รายเดือน, วิเคราะห์ breakdown, วงกลมอัตราสำเร็จ 96% (MTBF/MTTR), ฟีดแจ้งเตือน
2. **เครื่องจักร** — ตารางค้นหาได้, ป้ายสถานะ, ปุ่มลบ (มี dialog ยืนยัน), import/export
3. **แผน PM** — การ์ดแผน: ความถี่/ความสำคัญ/ความคืบหน้า/ครบกำหนด + ปุ่ม แก้ไข·Clone·พัก + แถบสถานะ auto-dispatch
4. **ใบสั่งงาน** — kanban 4 คอลัมน์ (รอ/กำลังทำ/เสร็จ/เกินกำหนด); คลิกการ์ดเปิด drawer: checklist, อัปโหลดรูป before/after, ความคิดเห็น
5. **ปฏิทิน** — มุมมองเดือน (พ.ศ.) แสดง event PM, เลื่อนเดือนได้
6. **รายงาน** — การ์ดส่งออก Excel/PDF 8 ประเภท

## พฤติกรรมสำคัญ (Behavior)
- **กฎ auto-dispatch:** แผนที่ครบกำหนดภายใน ≤ 5 วัน และไม่ได้พัก → สร้างใบสั่งงานอัตโนมัติเข้า
  คอลัมน์ "รอดำเนินการ" (หรือ "เกินกำหนด" ถ้าเลยวันแล้ว) — ดูฟังก์ชัน `autoDispatchDueWork()`
- **ปฏิทินแสดงปี พ.ศ.** (ค.ศ. + 543) เดือนเป็นภาษาไทย
- ป้ายภาษา: ป้ายไทยคู่กับศัพท์อังกฤษ (เช่น "แผน PM / PM Plans")

## หมายเหตุ
- RLS ในสคริปต์ตั้งไว้แบบง่าย (ผู้ที่ login แล้วเข้าถึงได้ทั้งหมด) — ปรับให้ละเอียดตาม role
  (manager/engineer/technician) ได้ภายหลังในคอลัมน์ `profiles.role`
- ถ้าต้องการ **realtime** (เห็นการเปลี่ยนแปลงของคนอื่นทันที) Supabase รองรับผ่าน
  `supabase.channel(...).on('postgres_changes', ...)` — เพิ่มทีหลังได้
- กรณีอยากใช้ **Firebase** แทน: โครงสร้างคอลเลกชันใช้ชื่อเดียวกันได้ แต่ต้องเขียน data layer ใหม่
  ตามรูปแบบ Firestore (โครง schema และตรรกะธุรกิจในเอกสารนี้ยังใช้อ้างอิงได้)
